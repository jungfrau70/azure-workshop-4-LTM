❯ Is a separate Landing Zone exercise necessary?

No, it is not. This workshop has already covered every layer that makes up a Landing Zone.

Landing Zone = the sum of all modules

Pre-work  → Platform Layer   : MG hierarchy + Korea policy
Module 1  → Governance Layer : RBAC + Policy + Resource Lock
Module 2  → Network Layer    : Hub-Spoke + NSG 3-tier
Module 3  → Identity Layer   : Managed Identity + Key Vault

Adding a separate "Landing Zone exercise" would simply duplicate work already completed.

In the workshop, when asked "Do you have Landing Zone design experience?", explaining it layer by layer — as you can now — is far more compelling than saying "I deployed it with one click using an ALZ template." The explanation "I built it step by step in the order MG → Policy → Hub-Spoke → Managed Identity, and I understand the role and interconnection of each layer" is much stronger.

The Landing Zone full picture added to the top of Scenario.md handles that contextual connection, so you can proceed to wrap up Module 7 without any additional exercises.
